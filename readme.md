# Описание движка 


## Технический стек
- PHP 7.4
- MariaDb 10.4
- JavaScript
- CSS
- Smarty (шаблонизатор)

## Структура проекта

### Основные компоненты
- `index.php` - Главный входной файл проекта
- `vendor/` - Автозагружаемые библиотеки (Composer)
- `site/` - Основная директория проекта

### Ключевые особенности

#### Отслеживание производительности
- Глобальная переменная `$ts` для измерения времени выполнения кода
- Возможность фиксации времени выполнения в любой точке кода через `$_SESSION['ts']`

#### Конфигурация
- `site/libs/config.php` - Основной файл конфигурации
- Содержит настройки подключения к БД и другие параметры
- Поддержка разных окружений (PROD/DEV) через `$config['release']`

#### Шаблонизация
- Использование Smarty 2.6.11 как основного шаблонизатора
- Все страницы формируются через шаблоны Smarty
- Настройки разделителей в Smarty:
  ```php
  $left_delimiter = '~~';
  $right_delimiter = '~';
  ```
- Основная директория шаблонов: `modules/`

##### Синтаксис Smarty (версия 2.6.11)

**Важно:** В этой старой версии Smarty все теги закрываются с использованием слэша `/`

**Вывод переменных:**
```smarty
~~$variable~
~~$array.key~
~~$object->property~
```

**Условия:**
```smarty
~~if $condition~
    content
~~/if~

~~if $condition~
    content
~~else~
    other content
~~/if~
```

**Циклы:**
```smarty
~~foreach from=$array item=item~
    ~~$item~
~~/foreach~

~~foreach from=$array item=item name=loopName~
    ~~if $smarty.foreach.loopName.first~первый элемент~~/if~
    ~~$item~
    ~~if $smarty.foreach.loopName.last~последний элемент~~/if~
~~/foreach~
```

**Использование в атрибутах HTML:**
```smarty
<div class="item~~if $condition~ active~~/if~">
```

**Специальные переменные в циклах:**
- `$smarty.foreach.loopName.first` - первая итерация
- `$smarty.foreach.loopName.last` - последняя итерация
- `$smarty.foreach.loopName.index` - текущий индекс (с 0)
- `$smarty.foreach.loopName.iteration` - номер итерации (с 1)

**Включение файлов:**
```smarty
~~include file="path/to/template.tpl"~
```

#### Работа с данными
- `site/libs/mysql.php` - Функции для работы с базой данных
- `site/functions.php` - Вспомогательные функции проекта

#### Модульная архитектура
- `site/classes/aModule.class.php` - Базовый класс для всех модулей
- Каждый модуль наследуется от базового класса
- Модули содержат свои шаблоны и логику

#### Маршрутизация
- `site/classes/router.class.php` - Класс для маршрутизации запросов
- Определяет направление запросов к соответствующим модулям
- Обрабатывает AJAX-запросы и сервисные команды
- Поддерживает параллельные группы модулей (например, market/)
- Проверки на:
  - Вызов sitemap.xml
  - Очистку сессий
  - AJAX-запросы
  - Математику корзины

#### Адаптивность
- Определение типа устройства через `detect_mobile()`
- Разделение шаблонов на:
  - `TPL = 'd'` - desktop версия
  - `TPL = 'm'` - мобильная версия

## Структура модулей

### Организация модулей

Все модули текущего разрабатываемого сайта находятся внутри организованной для этого папки site/modules/sfera/
- Модули организованы в директории (например, `site/modules/sfera/`)
- Каждый модуль содержит:
  - Класс модуля (`module_name.class.php`)
  - Шаблоны (`.tpl` файлы)
  - JavaScript/CSS файлы
  - Дополнительные ресурсы

### Вызов модулей
Модули вызываются через специальный синтаксис в шаблонах:
```smarty
~~mod path="sfera/" mod_name="menu" tpl="top_menu"~
```

Где:
- `path` - подпапка в основной директории modules
- `mod_name` - имя вызываемого модуля
- `tpl` - параметр, указывающий какой шаблон использовать

### Работа модулей
1. Контроллер модуля получает параметры через `$_SESSION[$session_name]`
2. Обрабатывает данные (например, делает запросы к БД)
3. Подготавливает данные для шаблона через `$_SESSION['smarty']->assign()`
4. Возвращает результат через `print ($_SESSION['smarty']->fetch())`

### Пример структуры модуля
```
site/modules/sfera/menu/
├── menu.class.php    # Контроллер модуля
├── top_menu.tpl      # Шаблон верхнего меню
└── other_templates.tpl
```

### Структура контроллера модуля

Каждый контроллер модуля:
1. Наследуется от базового класса `aModule`
2. Содержит обязательный метод `execute($arr)`
3. Может содержать дополнительные методы класса

Пример структуры контроллера:
```php
<?php 
include_once('site/classes/aModule.class.php');

class menu extends aModule
{
    // Обязательный метод, вызываемый при обращении к модулю
    function execute($arr)
    {
        // Обработка параметров из $arr
        // Логика модуля
        // Подготовка данных для шаблона
    }

    // Дополнительные методы класса
    function someMethod($params)
    {
        // Логика метода
    }
}
```

#### Особенности:
- Все модули наследуются от `aModule`
- Метод `execute($arr)` является точкой входа в модуль
- Параметры передаются в модуль через массив `$arr`
- Дополнительные методы вызываются через `$this->methodName()`
- Контроллер может содержать приватные методы для внутренней логики

### Вызов контроллеров через str: ресурс

Контроллеры могут быть вызваны не только из шаблонов, но и напрямую из PHP-кода через специальный механизм `str:` ресурса:

```php
print ($_SESSION['smarty']->fetch('str:'.$rout.'/showcase'));
```

#### Реализация в Smarty.class.php:
```php
$_SESSION['smarty'] = $smarty = new Smarty;

// Регистрация str: ресурса
$smarty->register_resource("str", array(
    "str_get_template",
    "str_get_timestamp",
    "str_get_secure",
    "str_get_trusted"
));

// Функция получения шаблона
function str_get_template ($tpl_name, &$tpl_source, &$smarty_obj)
{
    $arr = preg_split("/[\/]+/", $tpl_name);
    $rev = array_reverse($arr);
    $path_name = str_replace($rev[0], '', $tpl_name);
    $tpl_name = $rev[0];

    $tpl_source = "~~mod path='".$path_name."' mod_name='".$tpl_name."' params=\$post~";
    return true;
}

// Вспомогательные функции для str: ресурса
function str_get_timestamp($tpl_name, &$tpl_timestamp, &$smarty_obj)
{
    $tpl_timestamp = "123456";
    return true;
}

function str_get_secure($tpl_name, &$smarty_obj)
{
    return true;
}

function str_get_trusted($tpl_name, &$smarty_obj)
{
    // 
}
```

#### Преимущества использования str: ресурса:
1. Возможность вызова контроллеров напрямую из PHP-кода
2. Использование в AJAX-запросах
3. Формирование динамического контента
4. Гибкость в построении страниц
5. Возможность использования контроллеров как сервисных компонентов

#### Примеры использования:
- Формирование главной страницы
- Обработка AJAX-запросов
- Динамическая генерация контента
- Сервисные функции (например, корзина, поиск)
