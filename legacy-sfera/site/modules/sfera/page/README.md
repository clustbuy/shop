# Модуль статических страниц (Page)

## Описание

Модуль отображает статические страницы сайта из таблицы `pages`. Используется для страниц типа "О нас", "Контакты", "Доставка" и т.д.

## Архитектура

### Серверная часть (PHP)

**Файл:** `page.class.php`

#### Основные функции

1. **`execute($arr)`** - Главный метод модуля
   - Получает ID страницы из URL (например, `/page/32/` -> `$id = 32`)
   - Загружает данные страницы из таблицы `pages`
   - Если страница не найдена - показывает 404
   - Передает данные в шаблон

2. **`show404()`** - Показ страницы 404
   - Устанавливает HTTP заголовок 404
   - Отображает шаблон ошибки
   - Завершает выполнение

#### Получение данных страницы

```php
$items_query = [
    'from' => 'pages',
    'where' => ['id' => $page_id],
    'limit' => 1
];

$page_data = sel($items_query, true);
```

#### Структура данных страницы

```php
$page_data = [
    'id' => 32,
    'title' => 'О нас',
    'name' => 'about',
    'content' => '<p>HTML контент страницы</p>',
    'parent_id' => 0,
    'sort' => 1,
    'active' => 1
]
```

### Клиентская часть

**Файл:** `page.tpl`

#### Отображение

Страница обычно содержит:
- Заголовок (`$page.title`)
- HTML контент (`$page.content`)
- Breadcrumbs (хлебные крошки)
- Дополнительные блоки

## Использование

### URL структура

```
/page/{id}/
```

Пример: `/page/32/` - страница с ID 32

### Создание страницы

Страницы создаются через админ-панель или напрямую в БД:

```sql
INSERT INTO pages (title, name, content, active) 
VALUES ('О нас', 'about', '<p>Контент страницы</p>', 1);
```

## Файлы

- `page.class.php` - Серверная логика
- `page.tpl` - Шаблон страницы

## Зависимости

- Таблица `pages` - хранение страниц
- Функция `sel()` - для выборки данных

## Особенности

1. **Валидация:** Проверка существования страницы, 404 если не найдена
2. **HTML контент:** Поддержка HTML разметки в контенте
3. **Активность:** Отображение только активных страниц (`active = 1`)
4. **Иерархия:** Поддержка родительских страниц через `parent_id`

## Структура таблицы pages

```sql
CREATE TABLE pages (
    id INT PRIMARY KEY AUTO_INCREMENT,
    title VARCHAR(255),      -- Заголовок страницы
    name VARCHAR(255),        -- Системное имя
    content LONGTEXT,         -- HTML контент
    parent_id INT,            -- ID родительской страницы
    sort INT DEFAULT 0,       -- Порядок сортировки
    active TINYINT DEFAULT 1  -- Активность (1 - активна, 0 - неактивна)
)
```
