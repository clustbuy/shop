# Модуль страницы 404 (Error 404)

## Описание

Модуль обрабатывает ошибки 404 (страница не найдена) и логирует информацию о запросе для анализа.

## Архитектура

### Серверная часть (PHP)

**Файл:** `e404.class.php`

#### Основные функции

1. **`execute($arr)`** - Главный метод модуля
   - Логирует информацию о запросе в таблицу `log_404`
   - Сохраняет IP адрес запроса
   - Отображает шаблон страницы 404

#### Логирование

```php
insert('log_404', [
    'text' => json_encode($arr, JSON_UNESCAPED_UNICODE),
    'ip' => $this->get_ip()
]);
```

Сохраняемая информация:
- Полный массив параметров запроса (`$arr`)
- IP адрес пользователя
- Timestamp (через `insert()`)

### Клиентская часть

**Файл:** `e404.tpl`

#### Отображение

Страница 404 обычно содержит:
- Заголовок "404 - Страница не найдена"
- Сообщение об ошибке
- Ссылку на главную страницу
- Поиск по сайту

## Использование

### Вызов модуля

Модуль вызывается автоматически при ошибке 404 или вручную:

```php
http_response_code(404);
print ($_SESSION['smarty']->fetch('sfera/e404/e404.tpl'));
```

## Файлы

- `e404.class.php` - Серверная логика
- `e404.tpl` - Шаблон страницы 404
- `d.tpl` - Десктопная версия
- `m.tpl` - Мобильная версия

## Зависимости

- Таблица `log_404` - для логирования ошибок
- Функция `get_ip()` - для получения IP адреса

## Особенности

1. **Логирование:** Все 404 ошибки сохраняются в БД для анализа
2. **Информативность:** Сохранение полной информации о запросе
3. **Аналитика:** Возможность анализа популярных несуществующих URL

## Структура таблицы log_404

```sql
CREATE TABLE log_404 (
    id INT PRIMARY KEY AUTO_INCREMENT,
    text TEXT,           -- JSON с параметрами запроса
    ip VARCHAR(150),     -- IP адрес
    created DATETIME DEFAULT CURRENT_TIMESTAMP
)
```
